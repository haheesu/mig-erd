Backend Specification

RDB 마이그레이션
BE 명세 및 Endpoint 설계

이 문서는 백엔드 팀 기준의 실무 설계 문서입니다. 단순 CRUD 수준이 아니라 상태 전이, 비동기 실행, 연결 테스트, 매핑 검증, 실행 이력까지 포함해 1차 납품 가능한 수준으로 정리합니다.

도메인 경계

Connection Context

소스/타겟 연결정보 등록, 수정, 테스트를 담당합니다.

Migration Definition Context

작업 정의, 매핑 규칙, 동기화 옵션을 관리합니다.

Execution Context

실행 요청, 실행 잠금, 진행 상태, 결과 기록을 담당합니다.

상태 전이

draft설정 작성 중. 실행 불가
ready연결 테스트/매핑 검증 완료. 실행 가능
running실행 중. 동일 job 동시 실행 금지
failed최근 실행 실패 또는 검증 실패
archived운영 제외. 조회만 허용

핵심 설계 결정

동시 실행 정책

  • 같은 job_id는 동시에 하나만 실행
  • running 상태에서 재실행 요청 시 409 Conflict
  • DB 상태 전이와 워커 enqueue를 한 트랜잭션으로 묶음

검증 정책

  • 저장과 실행을 분리
  • 실행 전 반드시 source/target 연결 테스트 성공 필요
  • 매핑 필수값 누락 시 ready 전환 금지

Endpoint 설계

POST
/api/v1/migration/sources/test

RDB 소스 연결 테스트. 저장 전 사전 검증 용도입니다.

  • 입력: sourceType, host, port, databaseName, username, password, jdbcParams
  • 성공 시 DB 버전, 연결 지연시간, 스키마 조회 가능 여부 반환
{
  "sourceType": "mysql",
  "host": "mysql-prod.example.com",
  "port": 3306,
  "databaseName": "product_db",
  "username": "migration_user",
  "password": "******",
  "jdbcParams": {
    "useSSL": true
  }
}
POST
/api/v1/migration/targets/test

MongoDB 타겟 연결 테스트.

  • 입력: uri, databaseName
  • 성공 시 서버 버전, 쓰기 권한, 컬렉션 조회 가능 여부 반환
POST
/api/v1/migration/jobs

마이그레이션 작업 생성. source/target과 job, mapping을 한 번에 생성합니다.

  • 저장 후 기본 상태는 draft
  • 실패 시 source/target/job/mapping 전체 롤백
{
  "jobName": "USER_SERVICE_USERS",
  "source": {
    "sourceType": "mysql",
    "host": "mysql-prod.example.com",
    "port": 3306,
    "databaseName": "user_db",
    "username": "migration_user",
    "passwordSecret": "secret:mysql/prod/user-db"
  },
  "target": {
    "targetType": "mongodb",
    "uri": "mongodb://mongo-user:***@mongo-prod.example.com:27017",
    "databaseName": "user_db_mongo",
    "collectionName": "users"
  },
  "syncMode": "full",
  "scheduleType": "manual",
  "mappings": [
    {
      "targetCollectionName": "users",
      "sourceTable": "users",
      "sourceColumn": "id",
      "targetField": "_id",
      "targetType": "objectId",
      "isKeyField": true,
      "isPkMapping": true,
      "enabled": true,
      "sortOrder": 1
    }
  ]
}
GET
/api/v1/migration/jobs

마이그레이션 작업 목록 조회.

  • Query: status, sourceType, keyword, page, size
  • 기본 정렬: updatedAt desc
GET
/api/v1/migration/jobs/{jobId}

작업 상세 조회. source/target/mappings/latestExecution을 포함합니다.

PUT
/api/v1/migration/jobs/{jobId}

작업 수정. running 상태에서는 수정 금지입니다.

  • 수정 시 기존 mapping은 replace 전략 권장
  • 수정 성공 후 상태는 다시 draft 또는 ready 재판정
POST
/api/v1/migration/jobs/{jobId}/validate

실행 가능 상태 검증.

  • source 연결 테스트 결과
  • target 연결 테스트 결과
  • 필수 매핑 존재 여부
  • 중복 targetField 여부
POST
/api/v1/migration/jobs/{jobId}/activate

검증 통과 시 상태를 ready로 전환합니다.

POST
/api/v1/migration/jobs/{jobId}/run

마이그레이션 실행 요청. 비동기 워커로 enqueue 합니다.

  • 선행조건: status = ready
  • 성공 시 executionId 반환
  • DB row lock 또는 상태 업데이트 조건절로 중복 실행 차단
{
  "runMode": "manual",
  "requestedBy": "admin",
  "reason": "초기 적재 실행"
}
POST
/api/v1/migration/executions/{executionId}/stop

실행 중지 요청. 워커 cooperative cancel 방식 권장.

GET
/api/v1/migration/executions/{executionId}

실행 단건 조회. 진행률, read/write count, latest error를 포함합니다.

GET
/api/v1/migration/jobs/{jobId}/executions

작업별 실행 이력 목록 조회.

DELETE
/api/v1/migration/jobs/{jobId}

소프트 삭제 권장. 실제로는 archived 전환을 권장합니다.

주요 에러 코드

MIGRATION_SOURCE_CONNECT_FAILED

소스 DB 연결 실패

MIGRATION_TARGET_CONNECT_FAILED

MongoDB 연결 실패

MIGRATION_JOB_INVALID_STATE

현재 상태에서 허용되지 않는 작업 요청

MIGRATION_MAPPING_INVALID

매핑 누락 또는 중복 정의

MIGRATION_EXECUTION_ALREADY_RUNNING

동일 작업 실행 중복 요청

MIGRATION_JOB_NOT_FOUND

작업 미존재

운영 관점 권장사항

트랜잭션

  • 작업 생성/수정은 DB 트랜잭션 필수
  • 실행 요청은 상태 변경 + execution row 생성 + queue enqueue를 원자적으로 처리

보안

  • 패스워드는 절대 API 응답에 다시 노출하지 않음
  • 민감정보는 마스킹 로그만 허용
  • 실행/중지/삭제는 관리자 권한 필요
